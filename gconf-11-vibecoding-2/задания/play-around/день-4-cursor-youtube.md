# день 4: подкасты через MCP

сегодня учимся главному навыку vibe coding: **разворачивать процесс через cursor**.

---

## проблема которую решаем

у тебя есть подкаст или видео на 2 часа. времени нет. хочется выжимку под свой запрос.

решение: MCP который вытаскивает транскрипт из youtube → cursor его видит → ты спрашиваешь что нужно.

---

## вот ссылка на MCP

https://github.com/jkawamoto/mcp-youtube-transcript

это MCP который умеет вытаскивать транскрипты из youtube видео.

---

## но мы НЕ даём тебе пошаговую инструкцию

потому что главное — не настроить конкретный MCP. главное — научиться разбираться когда непонятно.

**паттерн который не работает:**
→ "не получается" → сдаюсь → иду спрашивать в чат / гуглить

**паттерн который работает:**
→ "вот что я хочу сделать" → даю cursor контекст → спрашиваю что делать → cursor ведёт шаг за шагом

---

## что делаем

### шаг 1: даём cursor ссылку и просим помочь

открой cursor и начни новый чат с агентом (cmd+L или ctrl+L).

напиши:

```
вот ссылка на MCP для youtube транскриптов:
https://github.com/jkawamoto/mcp-youtube-transcript

я хочу добавить этот MCP в свой cursor чтобы он мог вытаскивать транскрипты из видео.

у меня [mac/windows]. что мне нужно сделать? веди меня шаг за шагом.
```

### шаг 2: слушаем что cursor скажет

cursor посмотрит репозиторий (или попросит тебя скопировать оттуда README) и расскажет что делать.

скорее всего он скажет про:
- установку uv (менеджер пакетов)
- настройку mcp_config.json в cursor
- перезапуск cursor

### шаг 3: идём по шагам

cursor будет давать шаги. делай их. 

**если что-то непонятно:**
```
не понял этот шаг. объясни подробнее
```

**если ошибка:**
```
не получилось, вот ошибка: [вставь ошибку]. что делать?
```

**если кажется сложным:**
```
а можно проще? или есть другой способ?
```

**ключевое:** не уходи гуглить или спрашивать в чате. разворачивай процесс прямо здесь, с cursor.

### шаг 4: тестируем

когда MCP настроен, попробуй:

```
вытащи транскрипт из этого видео: https://www.youtube.com/watch?v=dQw4w9WgXcQ
```

(или вставь ссылку на любое другое видео)

если работает — поздравляю. если нет — спроси cursor почему не работает и что делать.

## что попробовать когда заработает

найди подкаст который давно хотел посмотреть, но не было времени.

```
вытащи транскрипт из [ссылка]

дай мне 5 ключевых идей из этого подкаста
```

```
я занимаюсь [твоя тема]. что из этого подкаста применимо к моей работе?
```

```
какие конкретные действия рекомендуют? составь список
```

---

## зачем именно так

1. **MCP меняются** — конкретная инструкция устареет через месяц. навык "спросить cursor" — нет.

2. **cursor знает контекст** — он видит что у тебя настроено, какая система, какие ошибки. гугл этого не знает.

3. **это и есть vibe coding** — ты не учишь синтаксис и команды. ты разговариваешь и получаешь результат.
