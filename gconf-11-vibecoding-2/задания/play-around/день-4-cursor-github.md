# день 3: cursor + github

сегодня получаем доступ к общей базе знаний gconf и делаем свой первый вклад.

---

## зачем это

github — это место где лежит весь контекст потока: задания, cursor rules, use cases, методология.

когда репозиторий открыт в cursor — ты можешь спрашивать cursor про любую часть gconf. и cursor будет отвечать, потому что видит все файлы.

а ещё ты можешь добавлять туда свои находки. и они станут частью общей базы знаний.

---

## что делаем

### шаг 1: устанавливаем GitHub Desktop

1. скачай с https://desktop.github.com
2. установи
3. войди в свой github аккаунт (или создай если нет)

GitHub Desktop — приложение которое делает работу с git простой. никакого терминала.

### шаг 2: клонируем репозиторий

1. открой https://github.com/matsako/gconf-public
2. зелёная кнопка **Code** → **Open with GitHub Desktop**
3. выбери куда сохранить (запомни путь!)
4. нажми Clone

репозиторий скачается к тебе на комп.

### шаг 3: открываем в cursor

File → Open Folder → выбери папку gconf-public (туда куда склонировал)

теперь cursor видит всю базу знаний.

### шаг 4: разговариваем с контекстом

попробуй спросить cursor:

```
что такое vibe coding? найди определение в этом репозитории
```

```
какие cursor rules есть в gconf? покажи примеры
```

```
как устроен framework для работы с AI?
```

```
найди ответ на мой вопрос про [что угодно из потока]
```

**мини-челлендж:** задай cursor вопрос про поток, который тебя реально интересует. запиши ответ.

### шаг 5: делаем свой вклад

теперь самое интересное — добавь что-то от себя.

в репозитории есть папка `community/` — туда можно добавлять свои находки.

**что можно добавить:**
- кейс "что, так можно было?" — что-то что ты попробовал и оно сработало
- свой cursor rule который тебе помогает
- заметку про инсайт с потока
- вопрос который хочется обсудить

**как добавить:**

1. **создай файл в cursor**
   
   попроси cursor:
   ```
   создай файл community/tvoy-username-case.md с шаблоном для кейса
   ```

2. **напиши что хочешь** (или попроси cursor помочь сформулировать)

3. **коммитим через GitHub Desktop**
   
   открой GitHub Desktop — он покажет твои изменения:
   - слева увидишь изменённые файлы
   - внизу напиши описание: "добавил свой кейс"
   - нажми **Commit to main**
   - нажми **Push origin** (кнопка сверху)

готово — твой файл теперь в общем репозитории!

---

## зачем github

1. **общий контекст** — cursor видит всё и отвечает на вопросы
2. **вклад** — ты добавляешь в базу, другие учатся на твоём опыте
3. **"кнопка сохранения"** — каждый коммит = точка к которой можно вернуться
4. **синхронизация** — доступ с любого устройства
