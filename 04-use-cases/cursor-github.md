# cursor + github

как работать с репозиториями через cursor.

---

## зачем это

- весь контекст проекта в одном месте
- cursor видит код, документацию, историю
- можно спрашивать про любую часть проекта
- "кнопка сохранения" для всего что делаешь

---

## базовое использование

1. склонируй репо: `git clone [url]`
2. открой папку в cursor
3. cursor автоматически видит все файлы

**что можно спрашивать:**
```
объясни структуру этого проекта
```
```
найди где определена функция X
```
```
как связаны файлы A и B?
```

---

## github как "сохранение"

каждый коммит = точка сохранения.

если что-то сломал — можно откатиться.

**базовые команды:**
```bash
git status          # что изменилось
git add .           # добавить все изменения
git commit -m "описание"  # сохранить
git push            # отправить на github
```

или проси cursor:
```
сделай коммит с описанием "добавил новую фичу"
```

---

## работа с gconf репо

добавь этот репозиторий к себе:

```bash
git clone https://github.com/matsako/gconf-public.git
```

откройте в cursor — теперь можешь спрашивать:
```
что такое framework в контексте gconf?
```
```
найди примеры coach prompt
```
```
какие есть use cases?
```

---

## обновление

мы добавляем новые материалы. чтобы получить:

```bash
cd gconf-public
git pull
```

---

## для своих проектов

1. создай репо на github.com
2. склонируй к себе
3. работай в cursor
4. коммить и пуш регулярно

cursor может помогать с git:
```
какие файлы я изменил?
сделай коммит всех изменений
```

*github = версионирование + бэкап + возможность делиться. cursor делает работу с ним простой.*
