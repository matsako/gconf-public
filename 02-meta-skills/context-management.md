# context management — как давать контекст AI

чем лучше контекст — тем лучше результат.

---

## почему это важно

AI не знает:
- кто ты
- что ты уже пробовал
- какие у тебя ограничения
- что для тебя "хорошо"

без контекста AI галлюцинирует и даёт generic ответы.

---

## что включать в контекст

**кто ты:**
```
я продакт-менеджер в стартапе, 3 года опыта, команда 5 человек
```

**что делаешь:**
```
запускаю новую фичу, дедлайн через 2 недели
```

**что уже пробовал:**
```
уже делал X и Y, не сработало потому что Z
```

**какие ограничения:**
```
бюджет $500, нет разработчика, только я и дизайнер
```

**что значит успех:**
```
хороший результат = 100 регистраций за первую неделю
```

---

## форматы контекста

**в начале разговора:**
```
контекст: я [кто], делаю [что], ограничения [какие].
задача: [что нужно]
```

**через файлы (в cursor):**
- положи релевантные документы в папку
- cursor автоматически их видит
- можешь явно указать: "посмотри файл X"

**через @ упоминания (в cursor):**
```
@filename.md — посмотри этот файл и ответь на основе него
```

---

## сколько контекста давать

**мало** — AI будет додумывать (плохо)
**много** — AI потеряется в деталях (тоже плохо)

правило: достаточно чтобы умный человек понял задачу.

---

## обновление контекста

контекст устаревает. напоминай:

```
апдейт: с прошлого раза изменилось X. теперь ситуация такая: Y
```

или начинай новый чат с актуальным контекстом.

---

## cursor-специфика

в cursor контекст = файлы в папке.

**лайфхаки:**
- держи README.md с описанием проекта
- называй файлы понятно (не "draft1.md", а "landing-page-copy.md")
- удаляй нерелевантное — меньше шума

---

*контекст — это 80% качества ответа. вкладывай в него время.*
